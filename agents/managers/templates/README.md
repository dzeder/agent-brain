# Manager Templates

Domain-specific managers extend `agents/managers/base-manager.md` and live in
**product repos**, not here. "Template" means: copy `base-manager.md` to the
product repo, rename it, and fill in the TODOs for that domain (project IDs,
worker roster, rubric weights, escalation thresholds, examples).

The brain repo only holds the base template and shared infrastructure. The
hiring process for any new manager follows `AGENT_PRODUCT_CYCLE.md` §11
Agent Hiring & Onboarding — every manager joins with a completed hiring
checklist, a registry entry, and a layer assignment.

See also: `AGENT_PRODUCT_CYCLE.md` §08 (Manager Agent Design content) and §11.
