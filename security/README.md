# Security

## Layout

| Path | What it is |
|------|------------|
| `stride-template.md` | The blank STRIDE worksheet — copy per capability |
| `threat-models/` | Completed STRIDE worksheets, one file per component (`coaching-loop.md` is the worked example) |

| `log-scrubbing/test-cases.md` | Quarterly re-test target (§04 obligation) |

## Known gaps tracked here

`CLAUDE.md` lists three architectural gaps. Status as of M3:

1. **Coaching loop injection risk** — partially closed by
   `coaching-review-checklist.md` (policy → checklist gate) and by
   `threat-models/coaching-loop.md` (STRIDE enumeration). Still open:
   schema-level enforcement of `coaching_review` signed output (M4
   PR-review agent + future `/schemas/managers/coaching-review/` schema)
   and runtime enforcement (Managed Agents beta verification at
   adoption).

2. **Oversight surface generated by the system being overseen** — open.
   Independent anomaly detector + honeypot task system not yet designed.
   Out of M3 scope; revisit at M6 (oversight wiring + autodream).

3. **Hard enforcement of layer authority rules** — open. Policy is in
   place across CEO/manager/worker prompts (`/agents/`); architectural
   enforcement awaits Managed Agents beta verification. Adoption-time
   artifact lives here when we have it.

## Cadence

- **STRIDE worksheets** — per initiative; revisited on every PR that
  changes agent capabilities (per `AGENT_PRODUCT_CYCLE.md` §04).
- **Injection corpus** — additive; new vectors added when (a) a new
  surface is introduced, (b) a real-world incident reveals a new class,
  (c) OWASP LLM Top 10 updates.
- **Coaching-review checklist** — runs per coaching event (manager
  responsibility) and per prompt-update PR sourced from coaching.
- **Log-scrubbing patterns** — quarterly re-test (§04 Data & Privacy);
  add patterns when a new credential type or PII surface enters the
  system.

See `AGENT_PRODUCT_CYCLE.md` §04 Security & Threat Modeling for the
threat-modeling cadence and §13 Adversarial & Safety Testing for the
red-team protocol.
