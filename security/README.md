# Security

What belongs in this directory:
- **STRIDE worksheets per agent capability and tool** — one threat model per
  capability, completed during §04 Security & Threat Modeling for each
  initiative. Use `stride-template.md` as the starting point.
- **Injection test corpus** — concrete injection vectors used to test
  prompt-injection defenses. See `injection-corpus/`.
- **Red-team prompt library** — adversarial prompts used during §13
  Adversarial & Safety Testing. See `evals/adversarial/`.

This directory is a **priority build item** per `AGENT_PRODUCT_CYCLE.md`
§Known Gaps. The three specific gaps that need to be addressed first:

1. **Coaching loop injection risk.** External input can flow into a worker
   output, into manager coaching, into a permanent prompt update. Today
   this is mitigated by policy (all coaching contradictions must be flagged,
   prompt updates go through the PR Anti-Drift gate) but not architecturally
   enforced. A coaching-review checklist for every coaching event is the
   first deliverable here.

2. **Oversight surface generated by the system being overseen.** Every
   digest and alert flows through agents that could be compromised. An
   independent anomaly detector and a honeypot task system are needed and
   not yet designed.

3. **Hard enforcement of layer authority rules.** "No agent can modify its
   own system prompt" is a policy constraint, not an architectural one. The
   Anthropic Managed Agents beta may or may not enforce it — verify at
   adoption and document the verification artifact here.

See `AGENT_PRODUCT_CYCLE.md` §04 Security & Threat Modeling for the threat
modeling cadence and §13 Adversarial & Safety Testing for the red-team
protocol.
