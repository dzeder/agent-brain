# Log Scrubbing

Regex patterns and test cases for redacting secrets, credentials, and
PII from agent traces, coaching diffs, and PR diffs. Required by
`AGENT_PRODUCT_CYCLE.md` §04 Data & Privacy Security.

## Files

- `patterns.yaml` — the regex pattern library, organized by category
  (API keys, OAuth/JWT, connection strings, financial PII, contact PII,
  internal infrastructure)
- `test-cases.md` — input strings with `should_match` expectations;
  the quarterly re-test target

## Consumers

| Consumer | What it scrubs | When |
|----------|----------------|------|
| Langfuse trace pipeline | All trace events (input, output, tool args, tool results) | Pre-storage |
| Coaching-diff CI check | Coaching note text generated by manager agents | Pre-delivery; high-severity matches REJECT the coaching event per `/security/coaching-review-checklist.md` |
| PR Anti-Drift gate | PR diffs (esp. prompt files, schema files) | Pre-merge; high-severity matches block the PR for human review |
| Agent self-output check | Worker `output` field before signing | Pre-emit (TODO; see worker-side note in §04 Prompt Injection Defense) |

## Severity levels

- **high**: REJECT / BLOCK on match. Caller (coaching CI, PR gate) must
  not let the matched content pass through.
- **medium**: FLAG / SCRUB on match. Replace with the redaction token
  and continue; alert if the match volume spikes (potential exfiltration
  attempt).

## Cadence

Quarterly re-test is a §04 obligation. Procedure in `test-cases.md`.
The PR review agent (Milestone 4) will own the automated check; until
then the test set is run manually via `rg -f patterns.yaml` style tooling.
